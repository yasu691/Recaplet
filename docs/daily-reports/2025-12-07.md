## 17:42 News.jsonがAndroidアプリに反映されない問題の修正

- 概要: GitHub ActionsでNews.jsonが更新されているのにAndroidアプリに反映されない問題を調査・修正した
- 詳細:
  - **問題の原因**:
    - RSSフィード処理に時間がかかりすぎて、GitHub Actionsが10分のタイムアウトでキャンセルされていた
    - タイムアウトするとNetlifyビルドフックが叩かれず、Androidアプリが参照する`https://recaplet.netlify.app/data/news.json`が更新されない
  - **実施した修正**:
    1. GitHub Actionsのタイムアウトを10分→30分に延長
    2. `scripts/generate-summaries.ts`に並列処理を導入
       - フィード単位での並列処理（Promise.allSettled使用）
       - 各フィード内のアイテム処理も並列化
       - 1つのフィードが失敗しても他のフィードは処理継続
    3. TypeScript型定義追加（FeedConfig interface）
  - **技術的ポイント**:
    - Promise.allを使わずPromise.allSettledを使用することで、一部失敗しても全体が止まらない堅牢な設計
    - 2段階の並列化（フィードレベル + アイテムレベル）で処理時間を大幅短縮
- 次のアクション: 変更をコミット・プッシュして、実際のGitHub Actionsで動作確認
