## 23:04 GitHub Pagesデプロイエラーの修正

- 概要: GitHub Pagesへのデプロイ時にReact Error #418（Hydration Error）と404エラーが発生していた問題を解決
- 詳細:
  - **問題**: `/Recaplet/` サブパスでの配信に対応していなかった
    - JavaScriptパスが `/_expo/` のままで、`/Recaplet/_expo/` になっていなかった
    - デプロイワークフローでsedによる手動パス修正を行っていたが、JavaScriptバンドル内のルーティングまで修正できていなかった
  - **解決策**:
    - `app.json` の `experiments.baseUrl` を `/Recaplet` に設定
    - `.github/workflows/deploy-gh-pages.yml` からsedによる手動パス修正を削除
  - **変更ファイル**:
    - app.json: `experiments.baseUrl: "/Recaplet"` を追加、`web.baseUrl` を削除
    - .github/workflows/deploy-gh-pages.yml: "Fix base path in HTML" ステップを削除
  - **検証結果**:
    - ビルドログに `Using (experimental) base path: /Recaplet` が表示され、設定が適用されたことを確認
    - HTMLファイル内の全てのパスが `/Recaplet/` プレフィックス付きで生成されることを確認
- 次のアクション: GitHubにプッシュして実際のデプロイで動作確認

## 11:01 データ取得の404エラーを修正

- 概要: アプリは読み込まれたが、news.jsonの取得で404エラーが発生していた問題を解決
- 詳細:
  - **問題**: Web版で `/data/news.json` にアクセスしようとして404エラー
    - `experiments.baseUrl` 設定により、正しいパスは `/Recaplet/data/news.json`
    - コード内では空文字列のbaseUrlを使っていたため、サブパスが考慮されていなかった
  - **解決策**:
    - `app/(tabs)/index.tsx` のfetch処理を修正
    - Web版でも `Constants.expoConfig.extra.router.origin` を使用してフルURLでアクセス
  - **変更ファイル**:
    - app/(tabs)/index.tsx: baseUrlの取得ロジックを変更（空文字列 → `router.origin`）
  - **検証結果**:
    - ビルド成功、`dist/data/news.json` が正しく配置されていることを確認
- 次のアクション: コミット・プッシュして本番環境で動作確認
